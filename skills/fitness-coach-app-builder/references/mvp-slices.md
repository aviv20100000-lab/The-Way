# MVP Slices

## Default First Slice

When the user says "build the app" and does not specify a narrower target, start here:

1. Auth with coach and client roles
2. Coach dashboard with client list
3. Client detail page
4. One assigned plan flow
5. One client progress submission flow
6. One coach feedback action

This slice proves the business loop without requiring the whole platform.

## What To Defer By Default

Defer these unless the user explicitly makes them a priority:

- AI analysis
- Payment systems
- Community or social feed
- Challenges and gamification
- Complex analytics dashboards
- Push notification systems
- Multiple overlapping progress modules

## Fast Selection Rules

### Choose workout-first when
- the coach sells training programs
- compliance and completion matter more than nutrition detail

### Choose nutrition-first when
- the coach's core value is meal guidance
- the user cares about meal logging and review cadence

### Choose progress-first when
- the business runs on body transformation updates
- weekly accountability is more important than program complexity

## Stability Checks Before Expanding

Do not expand the MVP until all of these are true:

1. Coach can create or assign the core item
2. Client can see it clearly
3. Client can submit a relevant update
4. Coach can review the latest update
5. Permission boundaries were checked once end to end
